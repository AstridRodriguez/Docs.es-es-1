### <a name="disposing-of-services"></a><span data-ttu-id="1f273-278">Eliminación de servicios</span><span class="sxs-lookup"><span data-stu-id="1f273-278">Disposing of services</span></span>

<span data-ttu-id="1f273-279">El contenedor llamará `Dispose` para `IDisposable` tipos lo crea.</span><span class="sxs-lookup"><span data-stu-id="1f273-279">The container will call `Dispose` for `IDisposable` types it creates.</span></span> <span data-ttu-id="1f273-280">Sin embargo, si agrega una instancia al contenedor usted mismo, lo no se eliminará.</span><span class="sxs-lookup"><span data-stu-id="1f273-280">However, if you add an instance to the container yourself, it will not be disposed.</span></span>

<span data-ttu-id="1f273-281">Ejemplo:</span><span class="sxs-lookup"><span data-stu-id="1f273-281">Example:</span></span>

```csharp
// Services implement IDisposable:
public class Service1 : IDisposable {}
public class Service2 : IDisposable {}
public class Service3 : IDisposable {}

public void ConfigureServices(IServiceCollection services)
{
    // container will create the instance(s) of these types and will dispose them
    services.AddScoped<Service1>();
    services.AddSingleton<Service2>();

    // container did not create instance so it will NOT dispose it
    services.AddSingleton<Service3>(new Service3());
    services.AddSingleton(new Service3());
}
```

> [!NOTE]
> <span data-ttu-id="1f273-282">En la versión 1.0, el contenedor llama a dispose en *todos los* `IDisposable` objetos, incluso aquellos que no ha creado.</span><span class="sxs-lookup"><span data-stu-id="1f273-282">In version 1.0, the container called dispose on *all* `IDisposable` objects, including those it did not create.</span></span>
