## <a name="authentication-vs-authorization"></a><span data-ttu-id="4a7b4-110">Autenticación frente a Autorización</span><span class="sxs-lookup"><span data-stu-id="4a7b4-110">Authentication vs. Authorization</span></span>

<span data-ttu-id="4a7b4-111">La autenticación es un proceso en el que un usuario proporciona credenciales que después se comparan con las almacenadas en un sistema operativo, base de datos, aplicación o recurso.</span><span class="sxs-lookup"><span data-stu-id="4a7b4-111">Authentication is a process in which a user provides credentials that are then compared to those stored in an operating system, database, app or resource.</span></span> <span data-ttu-id="4a7b4-112">Si coinciden, los usuarios se autentican correctamente y, después, pueden realizar las acciones para las que están autorizados durante un proceso de autorización.</span><span class="sxs-lookup"><span data-stu-id="4a7b4-112">If they match, users authenticate successfully, and can then perform actions that they're authorized for, during an authorization process.</span></span> <span data-ttu-id="4a7b4-113">La autorización se refiere al proceso que determina las acciones que un usuario puede realizar.</span><span class="sxs-lookup"><span data-stu-id="4a7b4-113">The authorization refers to the process that determines what a user is allowed to do.</span></span>

<span data-ttu-id="4a7b4-114">La autenticación también se puede considerar una manera de entrar en un espacio (como un servidor, base de datos, aplicación o recurso) mientras que la autorización es qué acciones puede realizar el usuario en qué objetos de ese espacio (servidor, base de datos o aplicación).</span><span class="sxs-lookup"><span data-stu-id="4a7b4-114">Another way to think of authentication is to consider it as a way to enter a space, such as a server, database, app or resource, while authorization is which actions the user can perform to which objects inside that space (server, database, or app).</span></span>
